# Исправление ошибок TypeScript и типов

## Проблема
Редактор показывает множество ошибок ("красных линий") из-за того, что TypeScript не знает о методах плагинов (`pathfinder`, `collectBlock` и т.д.), добавленных к объекту `bot`. Также есть проблемы с импортом `goals`.

## Решение
1.  **Обновить `api/bot/types.ts`**: Создать расширенный тип `ExtendedBot`, который включает описание всех плагинов. Это научит TypeScript понимать, что у бота есть методы `pathfinder.setGoal`, `autoEat.enable` и т.д.
2.  **Обновить модули**: Заменить использование `require` для `goals` на нормальный импорт или типизированную переменную. Использовать `ExtendedBot` вместо `mineflayer.Bot` в контроллерах.

## Шаги реализации
1.  **`api/bot/types.ts`**:
    -   Определить интерфейсы для плагинов (`Pathfinder`, `CollectBlock`, `AutoEat`, `PvP`).
    -   Экспортировать тип `ExtendedBot`, который наследует `mineflayer.Bot` и добавляет эти поля.
    -   Обновить `IBotController` для использования `ExtendedBot`.

2.  **`api/bot/index.ts`**:
    -   Импортировать `ExtendedBot`.
    -   Привести `this.bot` к `ExtendedBot`.

3.  **`api/bot/survival.ts`, `decision.ts`, `exploration.ts`**:
    -   Обновить импорты.
    -   Исправить получение `goals` (перенести `require` в начало или использовать import).
    -   Убрать `(bot as any)`, так как теперь типы будут известны.

Это устранит большинство конфликтов и "красных линий".
