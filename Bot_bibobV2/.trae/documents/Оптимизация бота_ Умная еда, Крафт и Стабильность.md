# Оптимизация поведения бота: Еда, Стабильность и Развитие

## Проблема
1.  **Кик с сервера**: Бот отключается при подборе еды (вероятно, из-за конфликта пакетов авто-еды и инвентаря).
2.  **Голод**: Бот не ест качественную еду, хотя жалуется на голод.
3.  **Пассивность**: Бот не крафтит и движется хаотично.

## Решение

### 1. Настройка системы питания (AutoEat)
В файле `api/bot/index.ts` будет произведена детальная настройка плагина `mineflayer-auto-eat`:
-   **Приоритеты**: Явно задать список лучшей еды (`golden_carrot`, `cooked_beef`, `cooked_porkchop`, `bread` и т.д.), чтобы он не игнорировал их.
-   **Порог срабатывания**: Установить `startAt: 16` (есть, когда шкала чуть упала), чтобы не доводить до критического состояния.
-   **Стабильность**: Добавить задержку перед едой, чтобы избежать спама пакетами сразу после подбора предмета (что вызывает кик).

### 2. Логика развития (Decision Engine)
В `api/bot/decision.ts` будет добавлена цепочка крафта:
-   Если есть **Бревна** -> Крафтить **Доски**.
-   Если есть **Доски** -> Крафтить **Верстак**.
-   Если есть **Верстак** -> Крафтить **Палки**.
Это первый шаг к созданию инструментов.

### 3. Структурированное движение (Exploration)
В `api/bot/exploration.ts` алгоритм случайного блуждания будет заменен на более осмысленный:
-   Бот будет выбирать точки не абсолютно случайно, а пытаясь исследовать новые чанки.
-   Будет добавлена проверка на "застревание" (если координаты не меняются долгое время).

## Шаги реализации
1.  **`api/bot/index.ts`**: Добавить конфигурацию `bot.autoEat.options` в событии `spawn`.
2.  **`api/bot/decision.ts`**: Реализовать функции `craftPlanks`, `craftCraftingTable`.
3.  **`api/bot/exploration.ts`**: Улучшить выбор цели движения.
