# Minecraftmod

**Набор экспериментальных Minecraft-ботов (написано с нуля).**

> Репозиторий содержит несколько автономных ботов и вспомогательных скриптов для Minecraft — проекты в разной степени готовности: от прототипов «с нуля» до ботов с собственной базой данных. Идея — дать основу, на которой можно быстро прототипировать, тестировать и улучшать поведение ботов в серверах Minecraft.

---

## Коротко — elevator pitch

`Minecraftmod` — это коллекция экспериментальных ботов для Minecraft, ориентированных на исследование поведения, взаимодействия с игроками и интеграции с внешними данными. Подойдёт разработчикам и энтузиастам, которые хотят:

* быстро запустить прототип NPC/бота;
* поэкспериментировать с логикой принятия решений и обучаемыми компонентами;
* понять, как связать игровую логику с базой данных/внешними сервисами.

---

## Что внутри (на момент создания)

* `Bot_BibobV1` — ранний прототип бота.
* `Bot_bibobV2` — вторая версия (развитие идеи и инфраструктуры).
* `Voyager-main` — экспериментальный модуль/проект.
* `main-ti.py` — вспомогательный Python-скрипт (интеграция/тесты).

> Структура и список файлов отражены в корне репозитория. (Язык проекта: JavaScript/TypeScript + небольшие вспомогательные скрипты на Python).

## Статус проекта

**Alpha / эксперимент.**

* Некоторые боты — минимально рабочие прототипы.
* Некоторые модули — заглушки или заготовки с TODO-комментариями.

## Требования

* Node.js (v16+ рекомендовано, v18+ предпочтительно)
* npm или yarn
* Python 3.10+ (только если вы используете `main-ti.py` или другие Python-утилиты)
* Доступ к Minecraft-серверу (реальному или локальному серверу для теста)
* (опционально) СУБД: SQLite / PostgreSQL / MongoDB — в зависимости от выбранного модуля

## Быстрый старт

1. Клонируйте репозиторий:

```bash
git clone https://github.com/mrbezarate/Minecraftmod.git
cd Minecraftmod
```

2. Установите зависимости (пример для Node.js модулей):

```bash
npm install
# или
# yarn install
```

3. Настройте конфигурацию для интересующего бота — скопируйте и отредактируйте `config.example.json` (если есть) в `config.json`.

4. Запустите нужный модуль/скрипт. Пример (если есть npm-скрипт):

```bash
npm run start --workspace=Bot_bibobV2
# или напрямую
node Bot_bibobV2/index.js
```

> Если бот использует базу данных или внешние API — проверьте переменные окружения и конфигурационные файлы перед запуском.

## Архитектура и подход

Репозиторий придерживается модульного подхода: каждый бот — отдельная директория с собственной логикой и, при необходимости, интерфейсом к базе данных.

Рекомендуемые практики для дальнейшей работы:

* выносить общий код (API-клиенты, хранилища, утилиты логирования) в отдельную `lib/` или `packages/` директорию;
* писать unit-тесты для логики принятия решений бота;
* оформлять конфигурацию через `.env` или централизованные JSON/YAML файлы;
* документировать API и формат базы данных (schema files).
